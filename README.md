## 📍 위치기반 근처 병원 대기 현황 조회, 비교 및 원격 예약 / 근처 약국 위치·대기 시간 조회 플랫폼

### 🏥 <위치 기반 주변 병원 검색, 대기 현황 조회, 비교 및 원격 예약 서비스> 

본 서비스는 사용자의 현재 위치를 기반으로 주변 병원을 조회하고,

환자의 주호소에 해당하는 진료과목에 실시간 환자 대기 수 확인 후 대기열이 적은 병원으로 예약 후 진료,

나아가 진료 기록 및 처방 등 관련 의료 정보들을 관리·열람할 수 있게 도와주는 의료 편의 플랫폼입니다.

### 💊 <근처 영업중인 약국 위치·대기 시간 조회>

진료 이후나 야간에 주변에 영업중인 약국을 확인한 후 위치 정보, 연락처 등의 정보를 조회합니다.

대기가 있을 시 대기 시간 안내를 통해 예상 대기 시간이 가장 짧은 약국을 이용할 수 있도록 도와줍니다.

## 📍 프로젝트 소개

이 프로젝트는 환자가 병원에 방문하기 전 주변 병원에 대기 인원이 얼마나 있는지 조회하고 비교해서 병원을 선택한 후

원격 대기 등록을 통해 예약할 수 있도록 하여 병원 방문 시 발생하는 무한 대기 현상을 줄이고

의료서비스 이용의 효율성을 극대화 하는 것을 목표로 합니다.

위치 기반 서비스(LBS)와 실시간 데이터 처리를 활용하여

프론트엔드와 백엔드 간의 데이터 통신을 통해 병원 대기 정보를 제공하고,

사용자별 병원 방문 이력을 안전하게 관리할 수 있도록 설계되었습니다.


## 🏥 주요 기능

- 사용자 위치 기반 주변 병원 조회

- 실시간 대기 현황 확인 (대기 인원, 예상 대기 시간 등)

- 병원 상세 정보 조회 (주소, 의료진 정보, 전화번호 등)

- 병원 진료과목 별 대기 시간 비교 후 의료 서비스 이용

- 위치 기반 주변 약국의 위치, 영업시간 등 조회

- 진료기록 저장·조회

- 진료 일정 관리

- 진료 기록 및 처방전 등 관련 문서 저장 및 열람

## 🎯 기대 효과

병원 무한 대기 현상 감소 (불 필요한 대기 시간 감소)

사용자 중심의 의료 서비스 제공 (주호소 관련 진료 과목에 대한 진료)

개인 의료 내역 관리로 인한 정보 제공 편의성 향상

## 💊 MediQ 서비스 소개

### 💡 프로젝트 배경
환자가 병원에 방문하기 전, 주변 병원의 대기 인원을 미리 조회하고 원격으로 대기 등록을 할 수 있다면 어떨까요? **MediQ**는 병원 방문 시 발생하는 고질적인 '무한 대기' 현상을 줄이고, 의료 서비스 이용의 효율성을 극대화하기 위해 탄생한 위치 기반 의료 편의 플랫폼입니다.

### 🩺 주요 기능
- **위치 기반 탐색**: 내 주변 가까운 병원 및 야간/휴일 영업 약국 정보 제공
- **스마트 예약 시스템**: 진료과목별 실시간 대기 인원 비교 및 원격 대기 등록
- **의료 기록 관리**: 개인별 진료 기록, 처방전, 관련 문서 안전 저장 및 열람
- **진료 대기 안내**: 원격 대기 등록 후 내 차례가 다가오면 실시간 푸시 알림 송신


## 🎯 백엔드 프로젝트 목표

단순한 기능 구현을 넘어, **실제 서비스 운영 환경**을 고려한 튼튼한 백엔드 시스템을 구축하는 것을 목표로 하였습니다.

- **LBS 기반 실시간 데이터 처리**: 프론트엔드와 지연 없는 통신을 통해 정확한 병원 대기 정보 제공
- **안전한 의료 정보 관리**: 사용자별 병원 방문 이력 및 민감한 데이터의 안전한 저장과 접근 제어
- **확장성 고려**: 추후 병원 연동 및 트래픽 증가에 대비하여 유연하게 대처할 수 있는 RESTful API 설계

## 🌿 프로젝트 기획 및 설계

### 🗺 시스템 아키텍처 (Target Architecture)
> 대규모 트래픽 발생 및 병원 실시간 예약 부하를 분산하기 위해 설계한 이상적인 타겟 아키텍처입니다.

<div align="center">
  <img width="558" alt="Image" src="https://github.com/user-attachments/assets/1d1d884a-49c2-42f6-90af-b4ba87f2c018" />
</div>

- **L4/L7 Load Balancer**: 예약 요청이 몰릴 때 다수의 서버 인스턴스로 트래픽 분산
- **Scale-out 구조**: 유연한 서버 확장을 위한 Stateless 애플리케이션 서버 구성
- **Database Replication**: 조회(Read)와 예약(Write) 부하 분산을 위한 Primary-Replica 구조
- **Redis Cache**: 잦은 조회가 발생하는 실시간 대기열 데이터를 캐싱하여 DB 병목 현상 방지
